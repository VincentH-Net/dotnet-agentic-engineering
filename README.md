# dotnet-agentic-engineering

Battle-tested agentic engineering for [.NET](https://dotnet.microsoft.com), [Aspire](https://aspire.dev), [Azure](https://azure.microsoft.com), [Fabric](https://www.microsoft.com/en-us/microsoft-fabric), [Orleans](https://learn.microsoft.com/en-us/dotnet/orleans) and [Uno Platform](https://platform.uno)

## What this is

Optimized agentic engineering workflows, setups and skills that I use for building real-world .NET applications.

Everything here has been used with real-world codebases - not generated with prompts and uploaded untested.

This repo exists to:
- filter true engineering value from untested slop and marketing noise in the agentic ecosystem
- fill agentic ecosystem gaps for specific .NET and Azure technologies
- extend the agentic ecosystem with specialized technology and pattern skills that capture best practices for distributed, cross-platform .NET applications

## What's included
Currently this repo includes technology-independant content and content specific for [.NET](https://dotnet.microsoft.com) and [Uno Platform](https://platform.uno).

## Contributing

Since I am limiting the content of this repo to only contain things that I can verify in real-world work, the best way to contribute is to register an issue `Consider including X` with a description of / link to what you have created or found as well as your experience using it (if any) - greatly appreciated!

- technology-independant agent workflows
- distributed .NET Orleans applications
- cross-platform Uno Platform apps
